PHP:

It is clear that PHP is the backbone of this project. It is being used to display pages, log users in and interact with database.

Database:

In database, there are 5 tables being created for this project:
  • Album: This table is used to display album info of the song, includes: album ID(auto increment, primary), album title, artist, genre and artworkPath.
  • Artist:This table is used to display artist info of the song, includes: artist ID(auto increment, primary) and artist name.
  • Genre: This table is used to display genre info of the song, includes: genre ID(auto increment, primary) and genrename.
  • Song: This table is used to display song info, includes: song ID(auto increment, primary), song title, artist, genre, song duration, musicPath, albumOrder and number of plays.
  • User: This is the only table that is not associated with songs. It includes user ID(auto increment, primary), firstName, lastName, email, password, signupDate and profilePic
Song informatinos are pre-programmed inside of database so the user cannot change it. However they can change the user table by signing up.

JavaScript:

Responsible for playing music audios, including previous/next song, pause/play and looping.
The way to do it is to create script.js that uses audio object to do basic music playing functions.
Inside nowPlayingBar.php, we use script.js to implement the UI(for example showing play/pause), as well as using Jquery(Ajax) to accomplish fetching song from database.
register.js is responsible for showing/hiding sign in / register option in register.phps.
